- [✔️] `store_template(user_id, base64_template)` function

- [✔️] Match score threshold set karna (e.g. score > 40 = match)

## MODULE 5 — Voting System
**Goal:** Sirf eligible voter vote kar sake, sirf schedule ke andar, sirf ek baar, with location.

### Vote Eligibility Checks (backend):
- [ ] Voter logged in hai? (session check)
- [ ] Aaj voting ka din hai? (schedule check — start <= now <= end)
- [ ] Voter pehle vote nahi de chuka? (hasVoted check)
- [ ] Vote karte waqt fingerprint dobara verify karo

### Location Capture:
- [ ] Frontend pe `navigator.geolocation.getCurrentPosition()` use karo
- [ ] Latitude, Longitude capture karo
- [ ] Reverse geocoding — coordinates se city/area naam nikalo
  - OpenStreetMap Nominatim API use karo (free, no key needed)
  - `https://nominatim.openstreetmap.org/reverse?lat=X&lon=Y&format=json`
- [ ] Vote ke saath location MongoDB mein store karo

### Vote Cast:
- [ ] `POST /api/vote` route
  - Fingerprint verify karo
  - Schedule check karo
  - hasVoted check karo
  - Location store karo (lat, lon, city, area)
  - candidates mein votes++ karo
  - voter ka hasVoted: True karo
  - voted_at timestamp store karo
- [ ] Vote page UI — candidate cards with party name + Vote button
- [ ] Vote karne ke baad confirmation screen
